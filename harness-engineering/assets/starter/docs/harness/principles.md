# Harness Principles

Last reviewed: {{DATE}}

## Golden Principles

- Keep repository knowledge local, versioned, and discoverable.
- Keep `AGENTS.md` short; use it as a map to deeper docs.
- Make important rules enforceable with tests, linters, type checks, or CI.
- Prefer clear, inspectable project structure over clever hidden behavior.
- Capture plans, decisions, and technical debt as repository artifacts.
- Treat repeated failures as evidence that the harness needs improvement.

## Project-Specific Principles

Add principles here as the project matures. Each principle should be specific
enough that a future agent can apply it without guessing.

## Promotion Rule

If a principle is violated more than once, consider promoting it into a
mechanical check or a template.
